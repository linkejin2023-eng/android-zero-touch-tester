# 自動化測試與手動測項 Gap 分析報告

## 1. 測試驗證邏輯與差距 (核心部分)

| 測試類別 | 自動化驗證手段 (現狀) | 與手動測試之差距 (Gap) |
| :--- | :--- | :--- |
| **相機、麥克風與鬧鐘** | 依賴 Logcat 或 Intent 傳送成功即判定為 PASS | **存在虛假 Pass 風險**。無法得知是否真的有影像產出、無 App 畫面、無存檔仍判定通過，或麥克風是否損壞，或鬧鐘是否實際響鈴。 |
| **顯示器** | 讀取系統權限與電源狀態 | **無光學驗證**。無法確認螢幕是否物理損壞或黑屏。 |
| **音訊/錄音** | [NEW] 檔案時間戳與大小驗證 (`find -mmin`) | **具備基礎物理驗證**。已確認檔案真實生成且有資料量，解決虛假 Pass。 |
| **感測器** | 區塊解析器 + 多點採樣 (T0/T1) | **具備動態驗證**。可偵測 X/Y/Z 軸數值是否有實質跳動。(目前暫停偵錯 SAR 干擾問題) |
| **WWAN (LTE)** | 關閉 WiFi 進階連線測試 (IP + Ping + YouTube HTTP) | **具備真實通訊驗證**。已確認數據可經由 SIM 卡進行端到端傳輸。 |
| **NFC** | [NEW] Power Cycle + 實體讀卡事件 (Logcat/Dumpsys) | **具備實體感應驗證**。已確認感應線圈能讀取到標籤 UID，並支援靜態貼附測試。 |
| **Touch / Buttons** | N/A (已移除) | **避免誤導**。由於 ADB 模擬會繞過 Driver，無法驗證硬體損壞，故已移除相關測項以提升報告可信度。 |

## 2. 物理連接缺口 (目前完全無法自動化項目)

*   **USB 週邊設備**：隨身碟 (Storage)、滑鼠 (Mouse)、USB-C 影像輸出 (DP) 的插拔與功能驗證。
*   **硬體介面**：底部擴充介面 (Extension Interface) 與 Empower 模組的實體對接測試。
*   **物理觸感**：實體按鍵的按壓回饋與彈力確認。

## 3. 近期與未來優化目標

*   **強化驗證機制**：加入檔案完整性檢查（如相機照片大小）與即時畫面截圖比對。
*   **動態感測驗證**：實作讀取感測器數值變化的邏輯，以應對感測器卡死問題。
*   **硬體通訊增強**：掃描藍牙週邊訊號強度，並進行初步的資料傳輸壓力測試。
*   **穩定性測試**：導入自動化開機與休眠循環，捕捉偶發性的系統崩潰。

## 4. 數據摘要

*   **手動測項總數**：115+ 條
*   **目前自動化涵蓋**：29 條核心測項
*   **測試執行效率**：已將上述核心測項從數小時人工縮短至 82 秒。
